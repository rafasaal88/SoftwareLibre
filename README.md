# <center>Tema 1: Introducción al  Software Libre

## 1. Definición de Software Libre
### Características del Sofware privativo (Antónimo del Software Libre)

<li> El comercializador impone las condiciones de uso.
<li> Si tiene errores, solo el productor puede arreglarlo.
<li> No se puede adaptar un programa a necesidades específicas.  
<li> Modelo de ingresos basado en prohibir el derecho de copia.

### Definición de Software Libre, tal y como fue concebio por Richard Stallman

<li> Libertad para ejecutar el programa en cualquier sitio, con cualquier propósito y para siempre.

<li> Libertad para estudiarlo y adaptarlo a nuestras necesidades. Esto exige el acceso al código fuente.

<li> Libertad de redistribución, de modo que se nos permita colaborar con vecinos y amigos.

<li> Libertad para mejorar el programa y publicar las mejoras. También exige el código fuente.

### Licencias
<p> Es el mecanismo que se utiliza para garantizar, de acuerdo con la legalidad vigente lo siguiente:
<li> Las 4 libertades de SL.
<li> Permisos para que el receptor del programa pueda ejercer esas libertades.
<li> Restricciones:
<li> <ul> Dar crédito a los autores originales en caso de redistribución.
<li> <ul> No pueden ir en contra de las libertades del SL.  

### Free software → no significa software gratuito
<li> Sin embargo:
<li> <ul> por la 3ª libertad, cualquiera que posea un programa
puede redistribuirlo sin pedir contraprestación
económica ni permiso.

<li> <ul> lo que hace que el coste de un programa tienda al coste
marginal de copia.

<li> Puede cobrarse por el valor añadido que supone:
<li> <ul> dar una cierta garantía
<li> <ul> selección, actualización y organización de un conjunto
de programas
<li> <ul> servicio técnico
<li> <ul> personalización del software, etc.


### Términos relacionados
<li> **Open Source Software (programas de fuente abierta):**  

<li><ul> Promovido por Eric Raymond y la Open Source Initiative.

<li><ul> Criticado por RIchard Stallman y la Free Software Foundation.

<li><ul> Hace énfasis en la disponibilidad del código fuente, no en la libertad.

<li><ul> Más políticamente aséptico.

<li> **Freeware (programas gratuitos)**

<li><ul> Normalmente se distribuyen en binario y se pueden obtener sin coste.

<li><ul> A veces se obtiene también permiso de redistribución, pero otras no, manteniendo sólo del sitio oficial mantenido a ese efecto.

<li><ul> Es habitual que estos programas se usen para promocionar otros programas.

<li> **Shareware**

<li><ul> Es un método de pago de distribución, ya que los programas generalmente sin fuentes, se puede copiar libremente, pero no usar continuadamente sin pagarlos.

<li><ul> La exigencia de pago puede estar incentivada por:

<li><ul><ul> funcionalidad limitada.

<li><ul><ul> mensajes molestos.

<li><ul><ul> una simple apelación a la moral del usuario.

<li><ul><ul> amenazas legales, etc.

<li> **Charityware, Careware**

<li><ul> Generalmente Shareware, pero uyo pago o aportación voluntaria es para una organización caritativa.

<li> **Dominio público**
<li><ul> El autor renuncia absolutamente a todos sus derechos.

<li><ul> Tiene que estar declarado explícitamente en el programa, ya que sí no se dice nada, el programa es propietario.

<li><ul> Si además se proporciona los fuentes, el programa es libre.

<li>**Copyleft**

<li><ul> Software libre cuya licencia obliga a que las modificaciones que se distribuyan sean también libres.

<li> **Propietario, Cerrado, No libre**

<li><ul> Software que no es libre ni de fuente abierta
